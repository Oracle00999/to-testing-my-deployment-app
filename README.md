# Portiq Node Test API

This project exists only to test Portiq, the deployment platform being built locally.

It is a small Express CRUD API used to verify Node.js deployment, environment variables, runtime logs, health checks, Postgres, Redis, MySQL, and MongoDB service attachment, and redeploys.

It intentionally has no Dockerfile so the platform should detect Node from `package.json` and generate one.

## Required Environment Variables

- `APP_NAME`: display name for the API
- `API_KEY`: required as the `x-api-key` header for `/notes` routes
- `DATABASE_URL`: Postgres connection URL generated by a Portiq Postgres service
- `REDIS_URL`: Redis connection URL generated by a Portiq Redis service
- `MYSQL_URL`: MySQL connection URL generated by a Portiq MySQL service
- `MONGODB_URI`: MongoDB connection URL generated by a Portiq MongoDB service
- `PORT`: optional locally, defaults to `3000`

## Deploy Settings

- Internal port: `3000`
- Build command: leave empty
- Start command: leave empty
- Dockerfile path: leave empty

## Local Run

```bash
npm install
APP_NAME="Testing Notes API" API_KEY="secret123" DATABASE_URL="postgresql://portiq:password@localhost:5500/portiq" REDIS_URL="redis://:password@localhost:5600" MYSQL_URL="mysql://portiq:password@localhost:5700/portiq" MONGODB_URI="mongodb://portiq:password@localhost:5800/portiq?authSource=admin" npm start
```

## Test Requests

```bash
curl http://localhost:3000/health
curl -H "x-api-key: secret123" http://localhost:3000/notes
curl -X POST http://localhost:3000/notes \
  -H "content-type: application/json" \
  -H "x-api-key: secret123" \
  -d '{"title":"Hello","body":"Created from curl"}'

curl -H "x-api-key: secret123" http://localhost:3000/users
curl -X POST http://localhost:3000/users \
  -H "content-type: application/json" \
  -H "x-api-key: secret123" \
  -d '{"name":"Ada Lovelace","email":"ada@example.com"}'

curl -X PUT http://localhost:3000/cache/greeting \
  -H "content-type: application/json" \
  -H "x-api-key: secret123" \
  -d '{"value":"Hello from Redis","expiresIn":300}'
curl -H "x-api-key: secret123" http://localhost:3000/cache/greeting
curl -X DELETE -H "x-api-key: secret123" http://localhost:3000/cache/greeting

curl -X POST http://localhost:3000/products \
  -H "content-type: application/json" \
  -H "x-api-key: secret123" \
  -d '{"name":"Portiq sticker","price":12.50}'
curl -H "x-api-key: secret123" http://localhost:3000/products
curl -H "x-api-key: secret123" http://localhost:3000/products/1

curl -X POST http://localhost:3000/events \
  -H "content-type: application/json" \
  -H "x-api-key: secret123" \
  -d '{"name":"deployment.completed","payload":{"project":"portiq-test"}}'
curl -H "x-api-key: secret123" http://localhost:3000/events
```

## Portiq Service Test

1. Add Postgres, Redis, MySQL, and MongoDB services to this project in Portiq.
2. Redeploy the app so all four generated service URLs reach the container.
3. Test `/users` to confirm the app can talk to Postgres.
4. Test `/cache/:key` to confirm the app can talk to Redis.
5. Test `/products` to confirm the app can talk to MySQL.
6. Test `/events` to confirm the app can talk to MongoDB.
