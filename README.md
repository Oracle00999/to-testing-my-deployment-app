# Portiq Node Test API

This project exists only to test Portiq, the deployment platform being built locally.

It is a small Express CRUD API used to verify Node.js deployment, environment variables, runtime logs, health checks, service attachment, and redeploys.

It intentionally has no Dockerfile so the platform should detect Node from `package.json` and generate one.

## Required Environment Variables

- `APP_NAME`: display name for the API
- `API_KEY`: required as the `x-api-key` header for `/notes` routes
- `DATABASE_URL`: Postgres connection URL generated by a Portiq Postgres service
- `PORT`: optional locally, defaults to `3000`

## Deploy Settings

- Internal port: `3000`
- Build command: leave empty
- Start command: leave empty
- Dockerfile path: leave empty

## Local Run

```bash
npm install
APP_NAME="Testing Notes API" API_KEY="secret123" DATABASE_URL="postgresql://portiq:password@localhost:5500/portiq" npm start
```

## Test Requests

```bash
curl http://localhost:3000/health
curl -H "x-api-key: secret123" http://localhost:3000/notes
curl -X POST http://localhost:3000/notes \
  -H "content-type: application/json" \
  -H "x-api-key: secret123" \
  -d '{"title":"Hello","body":"Created from curl"}'

curl -H "x-api-key: secret123" http://localhost:3000/users
curl -X POST http://localhost:3000/users \
  -H "content-type: application/json" \
  -H "x-api-key: secret123" \
  -d '{"name":"Ada Lovelace","email":"ada@example.com"}'
```

## Portiq Service Test

1. Import/deploy this repo once without a service to confirm it fails without `DATABASE_URL`.
2. Add a Postgres service in Portiq.
3. Redeploy the app so the generated `DATABASE_URL` reaches the container.
4. Test `/users` to confirm the app can talk to the attached Postgres container.
